# Трек 2 — федеративное обучение и дифференциальная приватность

Научная новизна ВКР: обучение рекомендательной модели на данных нескольких
площадок **без централизации сырых данных** (федеративное обучение) и с
формальной гарантией приватности (дифференциальная приватность), с
количественной оценкой компромисса «приватность ↔ качество».

## Идея

В кросс-платформенной постановке каждая площадка (локаль) — отдельный
«клиент». В датасете Trivago роль идентификатора площадки играет поле
`platform`. Данные клиента не покидают его периметр: клиент локально дообучает
копию глобальной последовательной модели (GRU4Rec) на своих сессиях, а сервер
усредняет веса (Federated Averaging, FedAvg). Это соответствует требованиям
152-ФЗ к минимизации передачи персональных данных.

Дифференциальная приватность добавляется на этапе локального обучения
(DP-SGD, Abadi et al., 2016): индивидуальные градиенты обрезаются по норме `C`,
к усреднённому градиенту добавляется гауссов шум `σ·C`. Бюджет приватности `ε`
оценивается RDP-аккаунтантом. Перебор `σ` даёт кривую trade-off.

## Запуск (Windows, GPU)

Быстрая проверка на синтетическом сэмпле в формате Trivago:

```powershell
python -m scripts.make_trivago_sample --out data\trivago_sample
python -m scripts.run_federated --data data\trivago_sample --rounds 5 --sigmas 1.0 2.0
```

Реальные данные (подвыборка — для приемлемого времени; DP-SGD медленный из-за
per-example градиентов):

```powershell
python -m scripts.run_federated --data data\trivago --stream --nrows 2000000 --rounds 20 --sigmas 0.5 1.0 2.0
python -m scripts.plot_federated
```

Результаты: `results\track2_federated.json`, `results\track2_modes.png`,
`results\track2_tradeoff.png`.

## Что измеряется

| Режим | Смысл |
|-------|-------|
| Centralized | централизованное обучение — верхняя граница качества |
| Federated | FedAvg по площадкам без приватности — цена децентрализации |
| Federated + DP (σ) | приватное обучение при разных уровнях шума → разные ε |

Ожидаемый результат (методологически значимый): федеративное обучение немного
уступает централизованному, а усиление приватности (рост σ → снижение ε)
монотонно снижает MRR — это и есть кривая компромисса «приватность ↔ качество».

## Параметры

- `--rounds` — число раундов FedAvg; `--local-epochs` — локальных эпох на раунд;
- `--clients-per-round` — сколько площадок участвует в раунде;
- `--min-sessions` — минимум сессий, чтобы площадка считалась клиентом;
- `--sigmas` — список уровней шума DP-SGD (0 ≈ без приватности, больше — приватнее);
- `--nrows` / `--stream` — ограничение и потоковое чтение больших данных.

## Примечание о производительности

DP-SGD вычисляет градиент по каждому примеру отдельно (per-example clipping),
поэтому он на порядок медленнее обычного шага. Для эксперимента используйте
подвыборку и умеренное число раундов; цель — получить форму кривой trade-off,
а не абсолютный максимум качества.
